# Веб UI для MMO в стиле Starsector — памятка проекта

Дизайн-система **OLDSECTOR**: веб-клиент космической MMO (вдохновлён Starsector, оригинальный дизайн).
Тон: нейтральный/военно-технический sci-fi, на английском в самом UI.

## ⚠️ ПРАВИЛО: визуализацию делать в ДВУХ моделях (бой + превью)
В проекте ДВА независимых пути рендера, и любая доработка визуала оружия/эффектов (вспышки, факелы, лучи, частицы, свет, муззл-флеши и т.п.) должна быть реализована в ОБОИХ:
1. **Бой** — WebGL-рендерер `battle-gl.js` + симуляция в `_initBattle/step` (DC). Настоящий PBR-свет, нормал-карты, динамические точечные источники (`B.parts` flash и т.д.).
2. **Refit / Edit-превью** — облегчённый 2D-canvas-оверлей поверх GL-корабля (`_beamSim`/`_paintBeam`, `_turretSim`, hull-showcase, факелы превью и пр.). Здесь НЕТ PBR-света по нормалям — эффект имитируется на 2D (аддитивные bloom'ы, градиенты), но визуально должен совпадать с боем (цвет/частота/форма).
Не забывать про обе модели: если поправил эффект только в бою — он не появится в превью, и наоборот. Пример: muzzle-flash лазера — в бою это динамический точечный свет (`_flash`/`B.parts`), в превью — пульсирующий 2D-bloom у дула в `_paintBeam` (синхронизация по `fl`=flicker Hz и `mode`).

## Файлы
- `Order New Dawn.dc.html` — основной Design Component (все экраны + логика боя). Это единственный entrypoint.
- `battle-gl.js` — WebGL-рендерер тактического боя (свои шейдеры, без зависимостей). Подключён в `<helmet>`. Экспортит `window.BattleGL.create(canvas)`; есть 2D-фолбэк.
- `textures/` — карты планет (8K/4K в корне, 1K-бэкап в `textures/low/`), `textures/sprites/` — спрайты кораблей/астероида + их normal maps (`ship.png/ship_n.png`, `destroyer.png/destroyer_n.png`, `asteroid.png/asteroid_n.png`).
- `support.js` — рантайм DC (НЕ трогать, генерируется).
- `hulls.json` — **единый конфиг геометрии корпусов** (тюнится тут): на каждый `hullTex` — `texture`, `aspect` (H/W), `engines` (сопла), `turrets` (точки выхода оружия), `shieldArc` (полуширина дуги щита в долях π). Все координаты — доли квада спрайта от ЦЕНТРА корпуса `[forward, lateral]` (forward>0=нос, <0=корма; lateral — поперёк). Грузится в `componentDidMount` через `fetch` и перезаписывает поле класса `HULLS` (которое держит запасную копию по умолчанию на случай провала загрузки). См. ключ `_README` внутри файла.

## Несколько корпусов кораблей (per-hull sprite)
Корабль в бою выбирает текстуру по полю `s.hullTex` ('ship' по умолчанию, 'dest' = destroyer). Чтобы добавить новый корпус:
1. Положить `textures/sprites/<key>.png` + `<key>_n.png`.
2. `battle-gl.js`: добавить пару в массив `srcs` (`['<key>D',...],['<key>N',...]`). Отрисовка корпуса и `drawHeat` уже выбирают текстуру по `s.hullTex` (`tex[k+'D/N/DW/DH']`).
3. DC `_loadBattleSprites`: загрузить изображения, `this._buildHullMask(img,'<key>')` (alpha-маска для коллизий/нагрева, хранится в `this._hullMasks[key]`), и запечь `<key>P/<key>E` через `_bakeLit` (для 2D-фолбэка). `_maskFor(s)` возвращает маску корпуса.
4. `hulls.json` + поле класса `HULLS[key] = { texture, aspect: H/W, engines, turrets, shieldArc }` — координаты в долях квада `[forward, lateral]` от центра (forward>0=нос). `engines` — сопла факелов (`_initBattle`/step, `hull.engines`); `turrets` — точки выхода снарядов (цикл `s._turI` в блоке огня, муззл = `s.x + body_fwd*fwW + body_lat*latW`); `shieldArc` — полуширина дуги щита (`s.shieldArc`, идёт в шейдер `uPower = shieldArc*deploy`, и масштабирует блокировку щита `1.2|1.15 * (shieldArc/0.66)`). Поворотные сопла: внешняя пара = `[0,1]` / `[n-2,n-1]`.
5. Прокинуть корпус в ростер: `bat.hullTex:'<key>'` (читается в `_initBattle` → opts `_mkShip`). Для Refit-селектора добавить и в `fleetDef`.
- **Destroyer «Aegis»** (`hullTex:'dest'`, 1177×580, 4 сопла) добавлен как 5-й развёрнутый корабль игрока в `roster` + `fleetDef`.
- **Frigate «Talon»** (`hullTex:'frig'`, 529×448) — 6-й развёрнутый корабль. ⚠️ **Ориентация арта**: рендер рисует ВСЕ корпуса с поворотом `s.angle+π`, т.е. нос ДОЛЖЕН смотреть ВЛЕВО на текстуре (как `ship`/`destroyer`). Исходный фригат смотрел носом ВПРАВО → текстуры зеркалятся по горизонтали при импорте (run_script: `ctx.scale(-1,1)`), причём у normal map дополнительно инвертируется КРАСНЫЙ канал (`R→255-R`, т.к. при зеркалировании X-компонента нормали меняет знак). Если новый арт смотрит носом вправо — повторить этот флип.

## Визуальный стиль
- Палитра: near-black космос `#06090e`, тёмные стальные панели, **акцент задаётся CSS-переменными** `--ac1 #f47b3e` / `--ac2 #ff9b5c` / `--acRGB 244,123,62` / `--atmoRGB`. Акцент МЕНЯЕТСЯ под «планету дня».
- Шрифты: `Chakra Petch` (заголовки/лейблы), `JetBrains Mono` (данные/цифры).
- Мотивы: скошенные углы (clip-path polygon), угловые скобки, hairline-сетки, янтарное свечение, лого-буква O в OLDSECTOR с пульсом + орбитальной планеткой.

## Экраны (роутинг по state.screen)
login → create → затем in-game c нижней командной панелью:
Character · Fleet · Refit · Inventory · Map · Codex (+ Colony справа, Exit). Плюс Research (из Codex), Station (док со станции через DOCK на nav), Battle.
- **Логин**: «планета дня» — при каждом обновлении страницы циклично меняется планета (14 тел) + акцент меню + цвет атмосферы (localStorage `oldsector_planet_idx`). Планета — попиксельный canvas-рендер с нормал-картой; render-loop ОСТАНАВЛИВАЕТСЯ при входе в игру (экономия FPS), возобновляется при Exit.
- **Экран загрузки**: показывается при старте, при Exit и **перед боем**. `enterBattle` сначала предзагружает все ассеты (`_loadBattleSprites`: диффуз+нормаль спрайтов, запечённые PBR-хуллы, hull-mask), и только потом переходит в `screen:'battle'`; в `_initBattle` цикл `revealWhenReady` держит лоадер пока `gl.ready` — нет подгрузки текстур в бою.
- **Refit**: статы корпуса справа-вверху схемы, небоевые HULL MODULES слева (тратят OP вместе с оружием).
- **Fleet**: ростер — единый источник `state.roster` (и для UI, и для спавна в бою). Каждый корабль: `deployed` (участвует в боях) + `strategy`/`tactic` + `bat` (боевой статблок). В досье: Deploy/Recall (`toggleDeploy`, минимум 1 в строю), сегментные селекторы COMBAT STRATEGY (`FLEET_STRATEGIES`) и FLEET TACTIC (`FLEET_TACTICS`) → `setShipDoctrine`. Стратегия масштабирует `desiredRange` при спавне (`STRAT_RANGE_MUL`); `_initBattle` собирает флот игрока из `roster.filter(deployed)`. Панель досье прокручивается (`min-height:0; overflow-y:auto`).
- **Codex**: уровень изучения 1–10 ромбами. **Character**: Reset Skill Points рядом со Skill Points.

## Тактический бой (battle-gl.js + логика в _initBattle/step в DC)
- Рендер: PBR-материал (Cook-Torrance, через uniform `uMetal`/`uRough`; **корабли metal 0.6 / rough 0.4**, **астероиды отдельный КАМЕННЫЙ материал metal 0.0 / rough 0.92** — неметаллический, очень шероховатый; alphaTest 0.1, как three.js meshStandardMaterial), нормал-карты, **1 направленный источник света** uLight + ambient (−30%). Нейтральный тинт корпуса (без командной окраски). **Порядок отрисовки в scene-FBO: астероиды → факелы двигателей (`drawFlames`, ДО корпуса, чтобы сопла были позади) → корпуса → лучи/трассеры → `drawPoints` (вспышки/искры/взрывы) → щиты.**
  - **Источник света** (battle-gl.js, drawSprite, ~стр.421): directional, только направление (ни позиции, ни ширины). Сейчас `uLight = (-0.4,-0.5,1.0)` — сверху-слева, ~57° над сценой. Раньше был `(-0.5,-0.62,0.62)` (скользящий ~38°): корабли, смотрящие навстречу, отличались от яркого до почти чёрного, т.к. при низком свете NoL сильно зависит от разворота. Поднятый Z это лечит. Интенсивность направленного `lightCol=3.0` (в шейдере FS_SPRITE), ambient `uAmb=0.42` для кораблей / 0.4 для астероида.
- Аддитивные эффекты: **сине-голубые факелы сопел** (газовая горелка под давлением: 2 слоя — тугой коллимированный бело-голубой `core` + широкий тёмно-синий факел; 5 сопел жёстко привязаны к корпусу). ⚠️ **Сопла считаются в ЭКРАННОМ пространстве так же, как рисуется спрайт** (Order New Dawn.dc.html ~стр.1992): мир анизотропен (`sx` множит на cssW, `sy` на cssH), поэтому позиция сопла = `SC + fwd*lon*dw + right*lat*dh` (dw=`radius*3.4*zoom`, dh=dw*566/1280, fwd=(cos a,sin a), right=(−sin a,cos a)), затем `toWX/toWY` обратно в мир; скорость выхлопа задаётся в px/кадр и переводится в мир делением на `zoom*cssW` / `zoom*cssH`. Это даёт жёсткую привязку при любом повороте (раньше считалось тригонометрией в мире → при повороте уплывало). Позиции сопел `NOZ5=[[-0.400,-0.260],[-0.400,-0.160],[-0.345,0.000],[-0.400,0.160],[-0.400,0.260]]` (доли квада [forward, lateral], lon<0=корма), сняты с арта корпуса. **Факелы рисуются ОТДЕЛЬНЫМ проходом `drawFlames` ДО корпуса** (battle-gl.js ~стр.790), чтобы сопла были позади корабля; начальный размер частиц core `r:1.53+0.72`, факел `r:2.34+1.62` (−10% от исходного); разброс core `sc1*0.045`, факел `sc2*0.198`, jitter `dh*0.045` (−10%). **Мерцающий голубой свет у двигательного блока** — короткоживущий (life 2) динамич. точечный `_flash` в центре сопел, цвет `[0.35,0.62,1.0]`, мерцание `flick=0.6+rnd*0.7`, яркость `li=0.66*flick`. Искры, файрбол-взрывы, трассеры, **тонкий красный лазерный луч** (Vindicator) с круглой мерцающей точкой попадания, которая случайно бродит по корпусу каждые 3–5с. **Вспышка света в точке попадания** (`_flash`): аддитивный спрайт (×1.4 размер) + настоящий динамич. точечный свет в FS_SPRITE (до 8, подсвечивает корпус+астероиды по нормал-карте; лазер ×0.333 через поле `li`). Нагрев корпуса (тлеющие метки). Контур силуэта пирата тонкой линией (тест).
  - **«Синий каустический блик»** (рябь/каустика в точке попадания по щиту) — рисуется в blit-проходе через scene-FBO (`FS_BLIT`), данные в `ripArr`. ⚠️ FBO имеет Y-up конвенцию, экранные коорд. Y-down → центр каустики уходит в шейдер с ФЛИПОМ: `ripArr[+1] = 1.0 - rcy/Hcss` (battle-gl.js ~стр.683). X нормируется шириной (`/Wcss`), Y и радиус — высотой (`/Hcss`). Debug-кольца (`__dbg.hits`) хранят экранные Y-down коорд. БЕЗ флипа — не путать пространства. Радиус `leadR = age*0.31*size/Hcss` (исходный 0.31).
  - Каустика от **снарядов** (Order New Dawn.dc.html ~стр.2023): точка попадания проецируется на КРОМКУ щита `wx=s.x+cos(ang)*shieldR, wy=s.y+sin(ang)*shieldR`, а НЕ берётся сырая позиция снаряда `p.x/p.y` — снаряд за кадр проваливается к центру корпуса, и блик оказывался в центре, а не на кромке.
  - **Преломление пространства при гибели корабля** (`_warp` → `B.warps`, читается в blit-проходе `FS_BLIT`, battle-gl.js ~стр.772): радиус волны теперь МИРОПРИВЯЗАН и масштабируется с зумом — `leadR = age * 0.1551 * B.zoom` (раньше был фикс. экранный `age*0.2325` → при зуме казался маленьким, при отдалении огромным; коэф. также ×3 по просьбе). Центр через `sx/sy` + Y-флип FBO; амплитуда `w.t*14.0`.
- **Нагрев корпуса**: макс. число тлеющих меток = поле класса `MAX_HEAT_MARKS` (сейчас 20) в начале `class Component`; используется в двух местах при push (попадание снаряда + `_addHeat`), при превышении `s.heat.shift()` удаляет самую старую (FIFO). Метки также гаснут по `hm.t`.
- ⚠️ Тестовые debug-кольца (красные/синие кружки) ОТКЛЮЧЕНЫ: `drawDebugRings` в battle-gl.js сразу `return;` (данные `__dbg` и текстовый HUD-отладчик работают). Вернуть — убрать `return;`.
- Поле боя 3×3 «мира» (WORLD=3). **Флоты спавнятся на РЕАЛЬНЫХ краях мира** (`P=(v)=>v*WORLD` в `_initBattle`: игрок слева ~x0.15, пираты справа ~x2.85, разброс по Y на всю высоту) и сами сближаются к центру для боя. Камера на входе в бой ОТДАЛЕНА на весь мир (`B.zoom = 1/B.world*1.02`, ~стр.1758), чтобы флоты на краях были видны. Камера: **drag-to-pan + wheel-zoom (до 9×)**, кнопка ◎ FOCUS следит за Vindicator.
- Коллизии по **alpha-маске текстуры** корабля (не радиус).
- Щит: дуга, **плавно разворачивается от носа 3–6с по классу** (Frigate 3 → Capital 6). Привязан к корпусу (рисуется в scene-FBO аддитивно, БЕЗ отдельного пост-прохода — иначе отрывался при панораме). Блокировка считается по `угол_луча+π` vs `shieldA`.
- **Лазер баланс**: self-flux = 10% урона; щит цели ВКЛ → 10% в hull + 90% в flux цели; щит ВЫКЛ → 90% в hull.
- **Overload**: flux достигает max → щит мгновенно off, ВСЕ системы (двигатели/оружие) off на 10с, корабль дрейфует по инерции, затем flux=0 и всё включается.
- Тест-кнопки внизу: LASER ON/OFF, PIRATE FIRE, PIRATE SHIELD, VIND SHIELD — синхронизируются с реальным состоянием при старте боя (в enterBattle). Сейчас в бою для теста по 1 кораблю на сторону (полные флоты закомментированы в _initBattle).
- Панели снизу: слева — выбранный свой корабль (hull/flux/статус), справа — ближайший пират (hull/flux). DISENGAGE — справа-вверху.
- ⚠️ WebGL-canvas НЕ попадает в скриншоты (html-to-image его не читает) — проверять бой только вживую; верификатор по бою пропускать.

## Технические заметки
- **ИЗОТРОПНЫЙ МИР (СДЕЛАНО).** Раньше `sx` множил на cssW, `sy` на cssH с одним zoom → мир растянут по горизонтали, окружность=эллипс, отсюда ~15 костылей `*aspect` / `/cssW`. Теперь **единый масштаб `worldScale(B)=B.zoom*Hcss` на ОБЕ оси** (battle-gl.js `sx/sy/worldScale` + asteroid-параллакс), центрирующие смещения `Wcss/2, Hcss/2`. Вертикаль не изменилась, горизонталь подтянута → корабли не растянуты, щиты — настоящие круги. **Правило DC-side**: каждая мировая длина `px/cssW` → `px/cssH` (hull-mask `_randomHullOffset/_hullHit`, heat `_addHeat`+inline UV, `shieldR`, `desiredRange`, collision push); все `*aspect`-компенсации (`vy` движения, `by/vy` пушек) УБРАНЫ; обратные проекции (клик-выбор `mx` получил множитель `r.width/r.height`; сопла `SCx/SCy/toWX/toWY/toWVx`) пересчитаны под `zoom*Hcss`. Хелперы `_toScreen` (мир→экран, как sx/sy; dmgNums) и `_screenRingPoint` (точка на щите: worldR=Rpx/cssH, screen-угол=мировому) переписаны; маркеры на ТОМ ЖЕ экранном радиусе — поведение сохранено. `_screenRingPoint` всё ещё принимает `cssW`, но не использует (вестигиальный). Кадрирование: мир квадрат 3×3, zoom `1/world*1.02` → флоты на screen-x ~303/970 из 1280, боковые поля ~21%. **Если поля пустоваты — далее: aspect-форма мира worldW=3·aspect×3** (прокинуть aspect в `_initBattle`, поправить клампы по worldW/worldH). 2D-фолбэк `_drawBattle` НЕ переведён (рисуется только если WebGL не создался).
- **Производительность боя (foundation под рост числа кораблей/пуль):** горячие коллекции `B.projs`/`B.parts`/`B.dmgNums`/`B.warps` + `s.heat`/`s.sHits` чистятся **swap-remove'ом** через хелпер `this._prune(arr, key, free?)` (без пофреймового `.filter()` → без GC-мусора; порядок не сохраняется — ок для аддитивных частиц). **Пули (`B.projs`) — на free-list пуле**: `_prune(B.projs,'life',B.projFree)` возвращает мёртвые пули в `B.projFree`, спавн через `this._acquire(B.projs, B.projFree, init)` берёт переработанный объект (форма пули идентична → рендерер БЕЗ изменений; init задаёт все 9 полей). Удаление пули — `p.life = 0` (висящих ссылок нет). Частицы (`B.parts`) НЕ пулятся: гетерогенны (flame/spark/flash/blast/ring, разные поля + флаги, читаются рендером ветвлением) и капятся `MAXP` — пулинг вслепую рискован. Поиск цели — по **квадрату расстояния** + один `sqrt`; тактика «Hunt Frigates» взвешивает цель по размеру корпуса. Мёртвый debug-силуэт (`if(false&&this.hullMask)`) удалён из battle-gl.js.
